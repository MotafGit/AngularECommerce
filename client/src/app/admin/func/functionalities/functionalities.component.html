Welcome to my fictitious eccommerce webApp
This application is being built with Angular19 Tailwind .Net9 SQLServer and Redis
Features:
  • - User can add items to the cart even if he isnt currently logged in. The cart is saved and tracked using the client local_storage. This type of carts are saved for 7 days and then deleted
  • - If the user logs in with a cart made while he wasnt logged, that cart will be given to the user and wont be ever deleted unless the user chooses to.
  • - If the user logs in and there is a different cart "in memory" then the one the user currently has, the user is gonna receive a pop up to choose if he wants to keep the previosly saved cart, or the new one.

    For example, if the user A is currently logged in with 4 items in the cart, and logs out, the cart will remain acessible, with those items. If this user switches account, upon logging, will receive the popUp to choose.

    Another example. if the user added items to the cart, and left them there, and comes back to the app a few days later from a different browser or device, and starts adding items to the cart and then logs in, it will be reminded of the previous cart, and given the choice to keep or replace the cart

  • - User can register with any email address, even fake ones.
  • - User can click on the items, see the description and update the cart from the individual item

    User can search items using the search bar (needs to click enter after typing (for now))

    User can sort by name or price ascending or descending

    User can filter by type and brands, for example if the user choose "trousers" only trousers will show, but if chooses "Trousers" and "Zara", only trousers from Zara shall show up

  • - User can add items, update and remove items from the cart
  • - User can see the cart, with the items, with the price of each item, the total per item * quantity, and the total cart
  • - User can checkout the cart, fill the form, choose shipping method, choose type of payment and check the summary of the previous 3 steps + the cart items and confirm the order.

    This feature only works on the front end for now.

  • - If you are reading you must have received an admin login.

    if you log with that email you are gonna have access to the Admin Panel

    At the moment, in this panel you'll be able to edit or add new products

    All fields are mandatory. The image provided must be from a secure url (https) and must be either jpg or png or webp

    the backend shall compact the image and turn it into .webp if it isnt already

  • - Most of the app if not the whole app, is responsive until "md" and (for now) some pages are even mobile responsive .
